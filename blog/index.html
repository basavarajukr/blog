About right
and wrong in
UI Design

Dwinawan Hariwijaya

March 16, 2021

It's been a few days since I made a design class. The concept is to do daily tasks, review and chat

One day a participant asked, "Mas I used to make a dashboard design like this, what do you think?" while showing the dashboard design that has been made.

After seeing the dashboard design, I think I want to say "This is wrong, it should be like this..."

But I immediately thought... this design is not an exact science, who has the right to determine whether a design like this is wrong or right?

After thinking for a while, I finally answered the question by giving another view.

Before answering the question, I first duplicated the design, then I changed some parts.

"For a dashboard like this, I will make the design like this" (while showing some design changes)

"The design you made is not wrong, the user can still use it. If you use your design, then the way the user uses it will be like this..., If you use my design, then the way the user uses it is like that..." (explains how Later the user will use the design he made and the one I made)

Maybe if I answered "Wrong" , he would not use the design again, even though it could be that the design would be suitable if the conditions were different.

In my opinion, design is not about right or wrong, but adapting to context and needs.
